# 局域网远程管理系统

## 一、系统概述

### 1.1 项目简介

本系统是一个基于C++和Qt框架开发的局域网内远程电脑管理工具，采用客户端-服务端(C/S)架构设计。管理员可以通过服务端程序统一管理局域网内的多台电脑，实现远程监控和软件部署。

### 1.2 主要功能

| 功能模块 | 功能描述 |
|---------|----------|
| 软硬件信息采集 | 远程获取电脑的CPU、内存、磁盘、网络等硬件配置信息 |
| 软件列表查询 | 获取远程电脑已安装的软件清单 |
| 软件批量分发 | 向多台电脑同时推送并静默安装软件 |
| 软件远程卸载 | 远程卸载指定电脑上的软件 |
| 实时状态监控 | 显示所有客户端的在线状态 |

### 1.5 UDP自动发现机制

系统支持客户端自动发现服务端，无需手动配置服务器地址：

- **广播端口**: UDP 8898
- **广播间隔**: 3秒
- **魔数标识**: `LANMGR_SERVER`
- **工作流程**:
  1. 服务端启动后每3秒向局域网广播自己的IP和TCP端口
  2. 客户端监听UDP广播，收到后自动连接服务端
  3. 连接断开后5秒自动重连

### 1.4 技术栈

- **开发语言**: C++ 17
- **GUI框架**: Qt 5.15.2
- **网络通信**: Qt Network (TCP Socket)
- **数据格式**: JSON
- **编译工具**: qmake + MinGW/MSVC

## 四、编译指南

### 4.1 准备工作

1. 安装Qt 5.15.2或更高版本
2. 确保Qt的bin目录和编译器目录在系统PATH中

### 4.2 使用MinGW编译

```powershell
# 设置环境变量 (根据实际安装路径调整)
$env:PATH = "C:\Qt5\5.15.2\mingw81_64\bin;C:\Qt5\Tools\mingw810_64\bin;" + $env:PATH

# 编译服务端
cd LanManager\Server
qmake Server.pro
mingw32-make -j4

# 编译客户端
cd ..\Client
qmake Client.pro
mingw32-make -j4
```

### 4.3 使用MSVC编译

```powershell
# 首先启动VS开发者命令行，然后设置Qt路径
$env:PATH = "C:\Qt5\5.15.2\msvc2019_64\bin;" + $env:PATH

# 编译服务端
cd LanManager\Server
qmake Server.pro
nmake

# 编译客户端
cd ..\Client
qmake Client.pro
nmake
```

### 4.4 编译输出

编译成功后，在 `LanManager/bin/` 目录下生成：

| 文件 | 说明 | 大小(约) |
|------|------|----------|
| LanServer.exe | 服务端程序（带GUI） | 125 KB |
| LanClient.exe | 客户端程序（控制台） | 87 KB |

1. 在客户端列表中**勾选**目标电脑

### 6.4 心跳机制

- **心跳间隔**: 5秒
- **超时时间**: 15秒

- **断线检测**: 服务端每5秒检查一次，超过15秒无心跳则断开连接

## 七、系统信息采集详解

### 7.1 采集项目

| 信息项 | 采集方式 | Windows API/方法 |
|--------|---------|------------------|
| 计算机名 | Windows API | `GetComputerNameW()` |
| 操作系统 | 注册表读取 | `HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion` |
| CPU信息 | CPUID指令 | `__cpuid()` + `GetSystemInfo()` |
| 内存信息 | Windows API | `GlobalMemoryStatusEx()` |
| 磁盘信息 | Qt API | `QStorageInfo::mountedVolumes()` |
| MAC地址 | Qt API | `QNetworkInterface::hardwareAddress()` |
| IP地址 | Qt API | `QNetworkInterface::addressEntries()` |

### 7.2 采集代码示例

```cpp
// CPU信息采集
QString SysInfo::getCpuInfo() {
    int cpuInfo[4] = {0};
    char cpuBrand[49] = {0};
    
    __cpuid(cpuInfo, 0x80000002);
    memcpy(cpuBrand, cpuInfo, sizeof(cpuInfo));
    __cpuid(cpuInfo, 0x80000003);
    memcpy(cpuBrand + 16, cpuInfo, sizeof(cpuInfo));
    __cpuid(cpuInfo, 0x80000004);
    memcpy(cpuBrand + 32, cpuInfo, sizeof(cpuInfo));
    
    SYSTEM_INFO sysInfo;
    GetSystemInfo(&sysInfo);
    
    return QString("%1 (%2 核)")
        .arg(QString::fromLatin1(cpuBrand).trimmed())
        .arg(sysInfo.dwNumberOfProcessors);
}
```

## 八、软件管理详解

### 8.1 软件列表获取

通过读取Windows注册表获取已安装软件：

| 注册表路径 | 说明 |
|-----------|------|
| `HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall` | 64位程序 |
| `HKLM\SOFTWARE\WOW6432Node\Microsoft\Windows\CurrentVersion\Uninstall` | 32位程序 |
| `HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall` | 当前用户安装 |

读取的注册表键值：
- `DisplayName` - 软件名称
- `DisplayVersion` - 版本号
- `Publisher` - 发布者
- `InstallDate` - 安装日期
- `InstallLocation` - 安装路径
- `UninstallString` - 卸载命令

### 8.2 静默安装

系统会根据安装包类型自动选择静默参数：

| 安装包类型 | 静默参数 | 示例 |
|-----------|---------|------|
| MSI | `/quiet /norestart` | `msiexec /i app.msi /quiet /norestart` |
| Inno Setup | `/VERYSILENT /SUPPRESSMSGBOXES /NORESTART` | `setup.exe /VERYSILENT` |
| NSIS | `/S` | `installer.exe /S` |
| InstallShield | `/s /v"/qn"` | `setup.exe /s /v"/qn"` |
| 通用 | `/S /silent /quiet` | `installer.exe /S` |

用户也可以在分发时手动指定参数。

### 8.3 静默卸载

卸载时会：
1. 解析注册表中的UninstallString
2. 自动添加静默参数（如`/quiet`、`/S`等）
3. 执行卸载命令并等待完成

## 九、文件传输机制

### 9.1 传输流程

```
服务端                              客户端
   │                                   │
   │  CMD_FILE_TRANSFER_START          │
   │  {fileName, fileSize, args}       │
   │──────────────────────────────────►│
   │                                   │  创建临时文件
   │  CMD_FILE_TRANSFER_ACK            │
   │  {success: true}                  │
   │◄──────────────────────────────────│
   │                                   │
   │  CMD_FILE_TRANSFER_DATA           │
   │  [64KB数据块]                     │
   │──────────────────────────────────►│  写入文件
   │         ... (重复N次) ...         │
   │                                   │
   │  CMD_FILE_TRANSFER_END            │
   │──────────────────────────────────►│
   │                                   │  校验文件
   │                                   │  执行安装
   │  CMD_INSTALL_RESPONSE             │
   │  {success, message}               │
   │◄──────────────────────────────────│
   │                                   │  删除临时文件
```

### 9.2 传输参数

- **分块大小**: 64KB
- **临时目录**: 系统临时目录 (`%TEMP%`)
- **超时时间**: 安装等待最长10分钟

### 10.2 手动部署服务端

1. 将 `LanServer.exe` 复制到目标机器
2. 确保Qt DLL可用（方式二选一）：
   - 将Qt DLL复制到exe同目录
   - 将Qt bin目录添加到系统PATH
3. 配置防火墙放行端口8899和8898
4. 运行程序并启动服务

**所需Qt DLL列表（MinGW版）：**
- Qt5Core.dll
- Qt5Gui.dll
- Qt5Widgets.dll
- Qt5Network.dll
- Qt5Svg.dll
- libgcc_s_seh-1.dll
- libstdc++-6.dll
- libwinpthread-1.dll
- platforms/qwindows.dll

### 10.3 手动部署客户端

1. 将 `LanClient.exe` 复制到目标机器
2. 确保Qt DLL可用（仅需Qt5Core.dll、Qt5Network.dll）
3. 以**管理员权限**运行（用于软件安装/卸载）
4. 可设置为开机自启动

**设置开机自启动：**
```powershell
# 将快捷方式放入启动文件夹
$WshShell = New-Object -comObject WScript.Shell
$Shortcut = $WshShell.CreateShortcut("$env:APPDATA\Microsoft\Windows\Start Menu\Programs\Startup\LanClient.lnk")
$Shortcut.TargetPath = "C:\Program Files\LanManager_Client\LanClient.exe"
$Shortcut.Save()
```

### 10.4 批量部署脚本

```powershell
# deploy_client.ps1 - 批量部署客户端

$ServerIP = "192.168.1.100"
$ClientPath = "\\server\share\LanClient"
$TargetPCs = @("PC-01", "PC-02", "PC-03")

foreach ($pc in $TargetPCs) {
    Write-Host "正在部署到 $pc ..."
    
    # 复制文件
    Copy-Item -Path "$ClientPath\*" -Destination "\\$pc\C$\LanClient\" -Recurse -Force
    
    # 创建计划任务（开机自启动）
    Invoke-Command -ComputerName $pc -ScriptBlock {
        param($ip)
        schtasks /create /tn "LanClient" /tr "C:\LanClient\LanClient.exe -s $ip" /sc onlogon /rl highest /f
    } -ArgumentList $ServerIP
    
    Write-Host "$pc 部署完成"
}
```

## 十一、故障排除

### 11.1 常见问题

**Q1: 客户端无法连接服务端**

排查步骤：
1. 确认服务端已启动并监听正确端口
2. 检查网络连通性：`ping 服务器IP`
3. 检查防火墙是否放行端口8899
4. 检查客户端参数是否正确

**Q2: 软件安装失败**

排查步骤：
1. 确认客户端以管理员权限运行
2. 检查安装包是否完整传输
3. 尝试手动指定静默参数
4. 查看客户端控制台输出

**Q3: 无法获取系统信息**

排查步骤：
1. 确认客户端以管理员权限运行
2. 检查是否有安全软件拦截

**Q4: 客户端频繁断线**

排查步骤：
1. 检查网络稳定性
2. 确认没有其他程序占用大量带宽
3. 检查是否有防火墙/安全软件干扰

### 11.2 日志分析

服务端日志格式：
```
[HH:MM:SS] 事件描述
```

常见日志信息：
- `服务器已启动,监听端口 8899` - 服务启动成功
- `客户端 xxx 已连接` - 新客户端连接
- `客户端 xxx 心跳超时,断开连接` - 心跳超时
- `文件传输完成,等待客户端安装` - 文件传输成功
- `客户端 xxx 安装成功/失败` - 安装结果
